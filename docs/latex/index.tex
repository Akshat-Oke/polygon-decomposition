\hypertarget{index_autotoc_md2}{}\doxysubsubsection{Format of a polygon file}\label{index_autotoc_md2}
For a polygon of n points, the file has n+1 lines of which the first line has the number of vertices and the remaining n lines have the vertices listed one on each line. A vertex on a line has x and coordinates separated by a space.

\hypertarget{index_autotoc_md9}{}\doxysubsection{Running the program}\label{index_autotoc_md9}
Compile the program\+: {\ttfamily g++ \mbox{\hyperlink{main_8cpp}{src/main.\+cpp}} \mbox{\hyperlink{dcel_8cpp}{src/dcel.\+cpp}} \mbox{\hyperlink{merge_8cpp}{src/merge.\+cpp}} -\/o main}

You can run a single file or a series of testcases in a folder. Specify the path to the file or folder as the command line argument.

Ex. {\ttfamily ./main spain.\+txt} will decompose the polygon in {\ttfamily spain.\+txt}. This will log the time taken in {\ttfamily timelog.\+txt} in microseconds. Ex. {\ttfamily ./main countries/} will decompose all polygons in {\ttfamily countries/} and log the time taken and number of vertices in {\ttfamily timelog.\+txt} in microseconds.\hypertarget{index_autotoc_md10}{}\doxysubsection{Output format}\label{index_autotoc_md10}
The program writes the decomposed (and merged) polgons to {\ttfamily \mbox{[}filename\mbox{]}\+\_\+merged.\+txt} with the number of polygons on the first line and for each polygon, the number of points in that polygon followed by its points as in the input format.

Decomposed polygons not merged are saved to {\ttfamily \mbox{[}filename\mbox{]}\+\_\+decomposition.\+txt}. Ex. {\ttfamily ./main countries/spain.\+txt} will save the decomposition (without merging) to {\ttfamily countries/spain\+\_\+decomposition.\+txt} and save the merged decomposition to {\ttfamily countries/spain\+\_\+merged.\+txt}
